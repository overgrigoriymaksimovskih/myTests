GitHub Flow стратегия
GitFlow стратегия
The Forking Workflow стратегия
Подробнее тут: 
https://javarush.com/groups/posts/2693-komandnaja-rabota-bez-putanicih-razbiraem-strategii-vetvlenija-v-gite

В зависимости от того какая стратегия используется на проекте так нужно работать.
Возможно будет не чистый тип какой то стратегии а смешанный...

1. Уставливаем гит (гит и гитхаб разные вещи ;))
2. В настройках контроля версий IDEA -> git указываем путь к файлу git.exe

3. Cоздаем в гитхабе токен для доступа (Settings/Developer Settings)
4. Используя сгенерированный токен логинимся через IDEA в GitHUB (Version Control -> GitHub)
5. Создаем репозиторий в GITHAB, копируем его ссылку

6. IDEA -> file new -> from version control. 
Создаем новую папку на компе, указываем ссылку на репозиторий (копировали выше)
Ставим галочку "git" чтобы настроился гит и связал нашу локальную папку и репозиторий по ссылке.
